你的服务器闲着也是闲着，不如让它自己干活

从 Lobechat 连接服务器说起，聊聊个人自动化的思路、工具和实践
March 9, 2026 · 1 min
Table of Contents
  • 今天的发现
  • 已经在跑的自动化
  • 个人自动化工具推荐
  • 自动化的正确姿势

今天的发现#

今天搭网站时，我用 Lobechat 连接了自己的云服务器。Lobechat 是一个开源的 AI 聊天应用，通过插件可以连接外部工具，包括服务器终端。

不过今天我没用它的自动化功能，而是手动复制命令执行。因为搭建阶段每一步都需要判断和调整，自动化反而不合适。

自动化的最佳时机，不是一开始，而是当你手动做过一遍、流程已经确定之后。

已经在跑的自动化#

今天建站过程中，GitHub Actions 一直在默默工作。每次 git push，GitHub 自动编译部署，2 分钟上线。这就是自动化最好的样子：设置一次，永久生效。

个人自动化工具推荐#

工具难度适合场景费用
cron⭐定时执行脚本，Linux 自带免费
GitHub Actions⭐⭐CI/CD、定时任务公开仓库免费
n8n⭐⭐可视化工作流，支持 AI 节点自部署免费
Lobechat + 插件⭐⭐AI 驱动的服务器操作开源免费
Zapier⭐连接 SaaS 服务免费额度有限

其中 n8n 特别值得关注——开源、可自部署、无限工作流。2025 年加入了 AI Workflow Builder，用自然语言就能生成工作流，GitHub 超 16 万 Star。

自动化的正确姿势#

  1. 先手动做一遍 — 理解每个环节
  2. 记录流程 — 哪些步骤是固定的
  3. 脚本化 — 写成 shell 脚本或 n8n 工作流
  4. 设触发条件 — 定时或事件触发
  5. 加监控通知 — 出问题第一时间知道

最重要的原则：自动化是为了省时间，不是为了炫技。
