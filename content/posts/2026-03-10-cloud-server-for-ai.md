用云服务器跑 AI 应用之前，我也不知道该怎么选配置。

买贵了浪费，买便宜了跑不动——折腾了一段时间之后，有了一些实际经验，整理出来给有类似需求的人参考。

---

## 我在跑什么

先说我的场景，方便对号入座：

- **OpenClaw AI 管家**：7×24 小时运行，每天自动搜索新闻、生成日报、推送企微，偶尔处理对话
- **SOL 量化交易系统**：每 15 分钟运行一次，fetch K线数据、计算特征、模型推理、Bybit API 下单
- **Hugo 博客**：本地构建，git push 自动部署到 GitHub Pages，服务器不承载博客流量

这两个应用同时在一台服务器上跑，互不干扰。

---

## 配置怎么选

**CPU**：2核够用。AI 管家和量化系统的 CPU 占用都不高，主要是等待 API 响应和处理文本，不是计算密集型任务。

**内存**：**2G 最低，4G 更稳**。OpenClaw 本身占用约 300-500MB，量化系统的模型推理（32K参数的轻量 Transformer）占用约 200MB，加上系统开销，2G 偶尔会触到上限。如果预算允许选 4G。

**硬盘**：20G 系统盘够用。量化系统的原始数据（K线、OI、资金费率）加起来约 30-50MB，训练集大一些但不需要放在服务器上。

**带宽**：1-3Mbps 够用。AI 管家的流量主要是 API 请求和推送，量化系统的 Bybit API 数据量也不大。

**地区**：我选的是**华南**（广州/深圳），原因是访问 Bybit API 延迟低，企微推送稳定。如果主要面向海外用户，可以选香港或新加坡节点。

## 我的实际配置

目前在用：**2核2G，40G SSD，3Mbps 带宽，按月付费**。

运行情况：
- OpenClaw AI 管家稳定运行，无内存溢出
- 量化系统每 15 分钟循环一次，平均耗时 8-12 秒
- 服务器负载通常在 10-20%，有充足余量

## 几个实际踩过的坑

**坑一：选了国外节点，访问国内 API 不稳定**

早期选了新加坡节点，结果调用某些国内 API 偶尔超时。换回华南之后问题消失。如果你的应用主要调用国内服务（企微、百度等），优先选国内节点。

**坑二：内存不够，进程被 OOM kill**

第一台服务器是 1G 内存，OpenClaw 偶尔因为内存不足被系统杀掉，需要手动重启。升级到 2G 后稳定了。

**坑三：按量计费账户欠费停机**

早期用按量计费，某天忘记充值，账户欠费，服务器自动停机，量化系统中断了好几个小时。换成包月之后就没有这个问题了。**跑量化系统建议包月，避免欠费风险。**

## 不同场景的配置建议

| 场景 | 推荐配置 | 备注 |
|------|------|------|
| 只跑 AI 管家 | 2核2G | 够用，留有余量 |
| AI 管家 + 量化系统 | 2核4G | 更稳，不用担心内存 |
| 加上模型训练 | 4核8G + GPU | 训练用 GPU 实例，推理用普通实例 |
| 只是建站 | 1核1G | 静态博客够用 |
