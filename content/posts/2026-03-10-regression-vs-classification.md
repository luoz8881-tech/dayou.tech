我的第一个 AI 量化模型，训练非常顺利。

Loss 稳定下降，验证集误差很低，各项指标看起来都不错。我满怀期待地把它接入回测系统，结果：

**策略收益跑输买入持有。**

这个结果让我困惑了很久。模型明明训练好了，为什么没用？

后来我才搞清楚，问题出在一个根本性的错误：**我用了回归模型，却期望它做分类的事。**

---

## 回归预测在做什么

我当时的思路是这样的：训练一个模型，预测未来 1 小时的收益率。如果预测值大于某个阈值，就做多；小于某个阈值，就做空。

听起来很合理。

但实际运行时，模型的输出长这样：

```
预测值：0.0012, -0.0003, 0.0008, -0.0001, 0.0005...
```

所有预测值都挤在 0 附近，几乎没有超过阈值的。

---

## 为什么会这样

问题在于**目标变量的分布**。

SOL 15 分钟 K 线的实际收益率，大约是这样分布的：

```
[-0.5%, +0.5%] 区间：覆盖约 85% 的数据
[-1%, +1%] 区间：覆盖约 97% 的数据
极端行情（>2%）：不到 1%
```

绝大多数 K 线的涨跌幅都很小。

对于回归模型来说，损失函数（MSE）是要最小化所有预测误差的平方和。模型发现了一个"最优策略"：**永远预测 0（或接近 0）。**

这样做的 MSE 极低，因为大部分数据的真实值本来就接近 0。但这个模型对交易毫无用处——它只是学会了"市场大部分时候不动"这个废话。

---

## 转向分类的逻辑

想清楚问题之后，我做了一个根本性的改变：**把回归问题变成分类问题。**

不再预测"未来收益率是多少"，而是预测"未来会涨、跌还是震荡"。

三个类别：
- **Up（涨）**：未来 4 小时收益率超过某个正阈值
- **Down（跌）**：未来 4 小时收益率低于某个负阈值  
- **Flat（震荡）**：介于两者之间

这个转变解决了几个问题：

**1. 模型不再能靠"预测 0"逃避学习**

分类模型用交叉熵损失，它必须给出每个类别的概率。如果总预测"Flat"，在真实涨跌行情里损失会很大，模型被迫去学习真正的方向性信号。

**2. 输出变成可操作的信号**

回归输出是一个连续数字，你不知道 0.0012 和 0.0015 的区别有多大意义。分类输出是概率：P(Up)=0.72，P(Down)=0.18，P(Flat)=0.10——这个信号清晰得多，超过置信度阈值就执行。

**3. 标签设计直接影响策略逻辑**

你可以主动设计"什么样的涨跌才算 Up"，这给了策略设计很大的灵活度。

---

## 标签边界的另一个坑

转向分类之后，我以为问题解决了。

但很快遇到了新的坑：**标签边界设多少？**

我最初用了固定值：收益率超过 0.8% 算 Up，低于 -0.3% 算 Down。

这导致了一个严重问题：**样本极度不均衡。**

在正常市场里，SOL 单根 15m K 线超过 0.8% 的情况很少。Up 类的样本数量远少于 Flat，模型根本学不到足够的 Up 信号，结果实盘几乎永远不触发做多条件。

解决方案：**用分位数自适应边界。**

不用固定值，而是用历史数据的 33% 和 67% 分位数作为边界。这样三个类别的样本数量大致均衡，模型能从每个类别学到足够的信息。

```python
# 固定边界（错误方式）
label = 'up' if future_return > 0.008 else ('down' if future_return < -0.003 else 'flat')

# 分位数自适应边界（正确方式）
p33 = df['future_return_4h'].quantile(0.33)
p67 = df['future_return_4h'].quantile(0.67)
label = 'up' if future_return > p67 else ('down' if future_return < p33 else 'flat')
```

## 多任务学习：两全其美

最终我用的是**多任务模型**，同时做分类和回归：

- **分类头**：预测方向（Up/Down/Flat），用于触发交易信号
- **回归头**：预测收益率幅度，用于辅助判断仓位大小

两个任务共享同一个 Transformer 编码器，损失函数是两者的加权和：

```
总损失 = 0.695 × 分类损失 + 0.305 × 回归损失
```

分类主导，回归辅助。这样既有清晰的方向信号，又保留了对幅度的感知。

---

## 小结

从回归到分类，这个转变背后是一个更根本的认知：

**模型优化的目标，必须和你真正想要的东西对齐。**

我想要的是"能赚钱的交易信号"，但我优化的是"MSE 最小"——这两件事根本不是一回事。

MSE 最小，意味着平均预测误差最小。但平均误差最小的模型，可以通过永远预测均值来实现，和交易没有任何关系。

这是 AI 量化最容易踩的第一个坑。踩进去过，才真正明白"目标对齐"是什么意思。
