有一件事困扰了我很久：每周一我要手动整理上周的数据，复制粘贴到表格里，然后发给相关的人。

这件事大概需要 40 分钟，每周都要做，每周都觉得浪费时间。

后来我用 **n8n** 把这件事变成了一个自动运行的工作流——现在它每周一早上 8 点自动完成，我只需要收到完成通知。

---

## n8n 是什么

n8n 是一个**开源的自动化工作流平台**。

你可以把它想象成一个"管道系统"：把不同的应用和服务通过管道连接起来，数据从一端流入，经过处理，从另一端流出。

```
触发器 → 处理节点 → 处理节点 → ... → 输出
```

例如：
- 收到邮件 → 提取关键信息 → 写入表格 → 发送通知
- 每天8点 → 抓取新闻 → AI摘要 → 推送到手机
- 表单提交 → 验证数据 → 创建任务 → 通知团队

**n8n 的核心优势是什么？**

跟 Zapier 这类工具比，n8n 最大的不同是可以**自己部署**。你的数据不经过第三方服务器，完全在自己的机器上运行。对于涉及敏感数据的场景，这一点非常重要。

另外，n8n 原生支持 AI 节点，可以直接把 ChatGPT、Claude 等模型接进工作流，实现智能化的自动化。

## n8n vs Zapier vs Make

这三个是目前最主流的自动化工具，选哪个取决于你的需求：

| 维度 | n8n | Zapier | Make |
|------|-----|--------|------|
| 费用（大量使用） | 低（自托管免费） | 贵 | 中等 |
| 数据安全 | ✅ 可自托管 | ❌ 数据过第三方 | ❌ 数据过第三方 |
| 学习曲线 | 中等 | 低 | 中等 |
| 集成数量 | 400+ | 6000+ | 1000+ |
| AI 原生支持 | ✅ 强 | 一般 | 一般 |
| 适合人群 | 技术友好型用户 | 纯非技术用户 | 非技术/轻技术 |

**我的建议：**
- 完全不懂技术、只需要简单自动化 → Zapier
- 数据安全有要求、或需要 AI 集成 → n8n
- 中间档 → Make

## 安装 n8n

最简单的方式是用 Docker，5 分钟搞定：

```bash
docker run -d \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  --restart unless-stopped \
  n8nio/n8n
```

安装完成后，浏览器访问 `http://你的服务器IP:5678`，注册账号，就进入了 n8n 界面。

> 💡 如果你已经有运行 OpenClaw 的云服务器，可以直接在同一台机器上装 n8n，节省成本。注意在防火墙规则里开放 5678 端口。

## 第一个工作流：每天定时发新闻摘要

我用这个案例带你走一遍完整流程，所有步骤都可以直接操作。

**目标：** 每天早上 9 点，自动从 RSS 获取科技新闻，用 AI 生成摘要，发送到邮件。

### 步骤一：新建工作流

进入 n8n 界面，点击左上角 **"+ New Workflow"**。

### 步骤二：添加触发器

点击画布中间的 **"+"**，搜索 **"Schedule"** 节点：
- Trigger Rule: Cron
- 表达式：`0 9 * * *`（每天9点）

### 步骤三：获取 RSS 内容

再点 **"+"**，搜索 **"RSS Feed Read"** 节点：
- URL：填入你想订阅的 RSS 地址
  - 科技：`https://sspai.com/feed`
  - 英文科技：`https://feeds.feedburner.com/TechCrunch`
- Items Count：填 `5`（每次抓5条）

### 步骤四：用 AI 生成摘要

添加 **"OpenAI"** 节点（或 Claude）：
- Operation: Message a Model
- Model: gpt-4o-mini（便宜够用）
- Prompt：
```
请将以下新闻标题和摘要整理成简洁的日报，每条100字以内，中文输出：

{{ $json.items.map(item => item.title + ': ' + item.contentSnippet).join('\n') }}
```

### 步骤五：发送邮件

添加 **"Gmail"** 或 **"Send Email"** 节点：
- To：你的邮件地址
- Subject：`每日科技简报 {{ $now.format('YYYY-MM-DD') }}`
- Content：`{{ $json.message.content }}`

### 步骤六：激活工作流

右上角点击 **"Active"** 开关，工作流开始运行。

## 10 个值得自动化的场景

这些都是我整理的高价值自动化场景，难度从低到高：

**入门级（不需要 AI）**
1. **表单提交自动入库** — Google Forms → Google Sheets → 邮件通知
2. **定时备份文件** — 每周自动把重要文件备份到云盘
3. **社交媒体定时发布** — 提前写好内容，定时发出

**进阶级（结合 AI）**
4. **RSS → AI摘要 → 推送** — 就是上面的案例
5. **邮件自动分类回复** — 收到邮件 → AI判断类型 → 自动回复
6. **竞品监控** — 定时抓取竞品页面 → AI分析变化 → 推送报告

**高阶级（多系统联动）**
7. **销售线索自动处理** — 表单提交 → CRM录入 → Slack通知 → 日历添加跟进提醒
8. **内容发布工作流** — 写好文章 → 自动发布博客 → 同步社交媒体 → 更新RSS
9. **财务数据汇总** — 定时抓取多平台数据 → 汇总表格 → 生成报告
10. **异常告警系统** — 监控服务状态 → 异常时立即通知

## n8n 的局限

诚实说，n8n 也有缺点：

- **学习曲线存在**：第一次上手要花几个小时才能顺畅
- **自托管需要维护**：服务器、更新、备份都要自己管
- **集成数量少于 Zapier**：6000+ vs 400+，有些冷门服务可能没有
- **调试复杂流程比较麻烦**：节点多的时候排错需要耐心

但对大多数人来说，这些都是一次性的学习成本，一旦跑通，就能持续节省大量时间。

## 和 OpenClaw 的配合

一个很自然的组合是 **OpenClaw + n8n**：

- OpenClaw 负责 **AI 决策层**：理解需求、搜集信息、生成内容
- n8n 负责 **数据管道层**：抓取数据、格式转换、多系统分发

比如我现在的一个工作流：
```
n8n 定时抓取多个数据源
    ↓
数据清洗、格式统一
    ↓
推送给 OpenClaw
    ↓
OpenClaw 分析、生成报告
    ↓
n8n 分发到多个渠道（邮件、企业微信、Telegram）
```

两个工具互补，覆盖了从数据获取到内容分发的完整链路。

## 小结

n8n 最大的价值不是"好用"，而是**让你意识到有多少时间被重复劳动浪费了**。

一旦你把第一件重复的事情自动化，就会开始想第二件、第三件。这种改变是累积的，半年后回头看，你会发现自己多出了大量时间做真正重要的事。

入门路径：
1. 先装好 n8n（Docker，5分钟）
2. 做一个最简单的工作流（定时发邮件）
3. 再找一件你每周重复做的事，把它自动化
